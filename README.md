## 🧭 What this is

ai-coding-principles is a set of Claude Code skills that help keep code work disciplined and reduce common AI coding mistakes. It gives you clear rules for writing, checking, and changing code with less drift and less guesswork.

This repo is for people who want more order in AI-assisted coding. It helps keep changes focused, code easier to read, and mistakes easier to spot.

## 💻 What you need

- A Windows PC
- Internet access
- A web browser
- Claude Code installed or ready to install
- Enough free space to save the files

The files are small and easy to keep on your computer. You do not need to set up a full development system to get started.

## 🪟 Install on Windows

After you download the release files, use these steps:

1. Open the folder where the file was saved.
2. If the file is in a ZIP folder, right-click it and choose Extract All.
3. Open the extracted folder.
4. Look for the skill files or package files inside the folder.
5. Copy the files into the Claude Code skills folder you use on your PC.
6. Restart Claude Code if it is already open.

If the release includes a setup file, double-click it and follow the on-screen steps. If it includes plain files, place them in the correct skills folder by using File Explorer.

## 🧩 What you get

This collection focuses on coding habits that help AI tools stay on track. It is built around simple rules that reduce common problems such as:

- code that grows too large
- repeated logic in many places
- changes that fix one issue and break another
- unclear file names and weak structure
- rushed edits that skip checks
- hidden assumptions in code
- code that is hard to test

The skills aim to keep each change small, clear, and easy to review.

## 🛠️ How to use it

Once the files are in place, Claude Code can use the skills during coding tasks. You can ask it to:

- keep changes small
- explain the plan before editing
- check for duplicate logic
- follow clean file structure
- avoid guessing when details are unclear
- review code for simple mistakes
- keep naming clear and consistent

You can also use the skills during reviews. They help you ask for a second look at the code before you save or share it.

## 📚 Skill areas

The collection is shaped by ideas often used in good software design and data design. It supports habits such as:

- clear boundaries between parts of a project
- simple data flow
- careful handling of state
- readable structure
- predictable changes
- testable code paths

These ideas help AI-generated code behave more like careful human work and less like a fast draft.

## 🧠 When to use it

Use these skills when you want to:

- build with Claude Code
- keep AI edits steady
- reduce messy code growth
- avoid repeated fixes
- keep your project easier to maintain
- keep work clear for later review

It fits well in small projects and larger ones. It helps when many changes happen in the same code base and you want less confusion.

## 🔍 What to expect

The skills do not write code for you by themselves. They guide how Claude Code thinks about the task. You should expect:

- more careful edits
- better questions before changes
- fewer wild changes
- clearer code structure
- more focus on the task you asked for

The best results come when you give short, clear requests and let the skills shape the response.

## 🧰 Good ways to use Claude Code with these skills

Try prompts like:

- Keep this change small and focused.
- Check for repeated logic before editing.
- Explain the file changes first.
- Avoid changing unrelated code.
- Use simple names and clear structure.
- Review the result for basic mistakes.

These prompts work well with the skills in this repo and help keep the output steady.
